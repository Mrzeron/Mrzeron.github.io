Flux 单向数据流

  1. View视图层
  2. Action(动作):视图层发出的消息
  3. Dispatcher(派发器):用来接收Actions执行回调
  4. Store(数据层):用来存放应用的状态，一旦发生变动，就提醒Views要更新页面

Flux的特点

  1. 用户访问 View
  2. View 发出用户的 Action
  3. Dispatcher 收到 Action，要求 Store 进行相应的更新
  4. Store 更新后，发出一个”change”事件
  5. View 收到”change”事件后，更新页面
  6. 数据总是”单向流动”，任何相邻的部分都不会发生数据的”双向流动”。这保证了流程的清晰。

新建todo_controller.jsx

采用React 的 controller view 模式”controller view”组件只用来保存状态，然后将其转发给子组件。

'use strict';
import React from 'react';
import Form_td from './form.jsx';//在form.jsx创建后引入 子组件层
import ListStore from '../stores/store.js';//在store.js创建后引入 store层
import ButtonActions from '../actions/addAction.js'; //在addAction.js创建后引入 action层

export default React.createClass(
    {
     getInitialState(){
         //初始化state数据
        return {
            items:ListStore.getAll()
        }
    },
    createNewItem(event){
                //todolist的点击添加事件
    var addvalue=this.refs.addval.refs.input.value; //input中的内容 可以利用refs取出子组件中的属性 this.refs是指本层的refs属性

        ButtonActions.addNewItem(addvalue);//将取出Input的数据通过调用action层的addNewItem方法传入action层
    },
    delItem(event){
            //todolist的点击删除
        let delIndex=event.target.dataset.index;//当前内容的index 通过给button绑定data-index={i}添加属性 然后在父组件通过event取出
        ButtonActions.delItem(delIndex);//将取出的当前index通过调用action层的delItem方法传入action层
    }
    ,
    componentDidMount(){//在数据渲染完成后 调用store层的方法并将_onChange函数传入
        ListStore.addChangeListener(this._onChange)
    },
    componentWillUnmount(){//在数据渲染完成后 调用store层的方法并将_onChange函数传入
        ListStore.removeChangeListener(this._onChange)
    }
    ,
    _onChange(){
            //在数据变化时更新state触发render
        this.setState({
            items:ListStore.getAll()  //调用store层的getall方法更新item
            })
    },
    render(){
        return(
            <Form_td  ref="addval"
            addBtn={this.createNewItem} items={this.state.items}
            delBtn={this.delItem}
            />
            //ref为子组件向父组件传递信息的钩子 addBtn为自定义属性将父组件的createNewItem方法传入 delBtn同理 items将父组件的state的items属性传入
        )
    }
})

新建dispatchers文件夹

新建AppDispatcher.js

Dispatcher 的作用是将 Action 派发到 Store、。你可以把它看作一个路由器，负责在 View 和 Store 之间，建立 Action 的正确传递路线。注意，Dispatcher 只能有一个，而且是全局的。

安装flux cnpm i flux -D;
import ListStore from '../stores/store.js';//引入store层准备把数据传入store层
var AppDispatcher = new Dispatcher(); //创建一个Dispatcher实例
import {Dispatcher} from 'flux';        //引入flux
 AppDispatcher.register(action=>{ //在Dispatcher实例中注册函数并将参数action传入
    switch(action.actionType){ //判断参数类型 类型为action层中设置根据不同类型做不同操作
        case 'ADD_NEW_ITEM':
        ListStore.addNewItemHandle(action.text);//调用store层的addNewItemHandle方法并将参数(新增item项)传入store层
        ListStore.emitChange();//触发state的刷新
        break;
         case 'DEL_ITEM':
           ListStore.delItemHandle(action.index);//调用store层的delItemHandle方法并将参数(要删除item项的索引)传入store层
           ListStore.emitChange();//触发state的刷新
        break;
    }
})
export default AppDispatcher;

新建stores文件夹 store层

Store 保存整个应用的状态。它的角色有点像 MVC 架构之中的Model

新建store.js

import {EventEmitter} from 'events'; //引入nodejs的事件对象
export default Object.assign({},EventEmitter.prototype,{
    items:[],//定义一个空数组用来保存新添加的li
    getAll(){
        return this.items //该方法返回items属性
    },
    addNewItemHandle(text){
        this.items.push(text); //用来将V层的数据添加进数组
    },
    delItemHandle(index){
        this.items.splice(index,1); //用来将V层的数据从数组里删除
    }
    ,
    emitChange(){
        this.emit('change'); //该方法会调用change方法
    },
    addChangeListener(callback){
        this.on('change',callback) //该方法对change方法会监听change事件回调函数调用时会刷新数据
    },
    removeChangeListener(callback){
        this.removeListener('change',callback);//该方法会移除change事件的事件监听并执行回调
    }
})//将EventEmitter对象与传入的对象merge
